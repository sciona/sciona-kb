## Key Insight

The load-bearing idea is the pipeline topology: Base Pair extraction creates the representation needed by Target-weighted Loss, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Base Pair extraction :red_circle:

This stage parse the raw mRNA string representations through the ViennaRNA package to extract the Base Pair Probability (BPP) connection matrices.

### Tensor formatting :red_circle:

This stage stack the 1D sequence and 2D BPP matrices into compatible PyTorch tensors (numpy).

### GCN Layer :warning:

This stage process the 2D BPP matrices through a Graph Convolutional Network (GCN) to allow structural nodes to pass chemical information to their bonded neighbors (PyTorch Geometric).

**Bound atom:** `sciona.atoms.dl.graph.adjacency_smoothing`

### RNN Layer :red_circle:

This stage process the 1D sequence strings through a Bidirectional GRU to capture the linear sequence context (PyTorch).

### Target-weighted Loss :white_check_mark:

This stage combine the embeddings and apply a specialized loss function during backpropagation that heavily penalizes errors on sequences with high signal-to-noise ratios, ignoring mathematically noisy lab readings.

**Bound atom:** `sciona.atoms.dl.loss.weighted_bce_loss`
