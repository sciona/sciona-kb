## Pipeline Stages

### Streaming imputation :white_check_mark:

This stage maintain a rolling state of previous row values to apply forward-filling for missing data dynamically (numpy).

**Bound atom:** `sciona.atoms.time_series_features.forward_fill`

### Action space mapping :red_circle:

This stage convert the single utility target into 5 distinct binary targets based on different time horizons.

### Feature dimensionality reduction :red_circle:

This stage train a custom autoencoder to project 130 features into a compressed space, then concatenate with original features (PyTorch).

### Multi-head MLP :red_circle:

This stage train a deep MLP with bottleneck layers and a custom utility-based loss function on the concatenated features (PyTorch).

### Streaming inference :red_circle:

This stage ingest data row-by-row via the competition API, update state arrays, and predict the binary action.
