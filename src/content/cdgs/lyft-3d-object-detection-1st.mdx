## Key Insight

The load-bearing idea is the pipeline topology: Voxelization creates the representation needed by Box-NMS, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Voxelization :red_circle:

This stage discretize the continuous LiDAR point cloud into a structured 3D grid of voxels (custom C++ / numpy).

### PointPillars encoding :red_circle:

This stage project the voxels vertically to convert the 3D space into a dense 2D pseudo-image (Bird's Eye View) (PyTorch).

### Multi-modal fusion :red_circle:

This stage project the 2D bounding boxes from camera images onto the 3D LiDAR space to "paint" the point cloud with RGB features (OpenCV).

### 2D CNN Backbone :red_circle:

This stage pass the BEV pseudo-image through a standard 2D CNN (ResNet/FPN) to predict 3D bounding box dimensions, rotation, and class (PyTorch).

### Box-NMS :red_circle:

This stage apply Non-Maximum Suppression specifically designed for rotated 3D boxes to drop overlapping predictions (numpy).
