## Key Insight

The load-bearing idea is the pipeline topology: BSON Chunking creates the representation needed by Test-Time Augmentation (TTA), so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### BSON Chunking :red_circle:

This stage parse the massive 50GB BSON file line-by-line without loading into RAM, decoding images to JPEG and writing to a lightning-fast LMDB database (PyMongo/LMDB).

### Multi-GPU Data Loader :red_circle:

This stage write a custom multi-threaded data loader utilizing PyTorch's DistributedDataParallel to constantly feed 8 GPUs without I/O bottlenecks (PyTorch).

### CNN Backbone :red_circle:

This stage train ResNet50 and InceptionV3 backbones utilizing a heavily tuned learning rate scheduler (PyTorch).

### Hierarchical Classification Head :red_circle:

This stage replace the standard flat softmax with a hierarchical softmax predicting Category -> SubCategory -> Item, reducing the vast 5000-class search space (custom PyTorch logic).

### Test-Time Augmentation (TTA) :white_check_mark:

This stage run 4x spatial crops on the test images, sum the log-probabilities, and select the argmax class.
