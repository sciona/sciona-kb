## Key Insight

The load-bearing idea is the pipeline topology: Linguistic Feature Extraction creates the representation needed by Final Regression, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Linguistic Feature Extraction :red_circle:

This stage process text to extract classic readability metrics such as the Flesch reading ease score and the SMOG index as structural metadata (textstat).

### Pairwise Construction :red_circle:

This stage construct training samples by pairing different text excerpts against each other rather than treating them as isolated regression targets.

### Bradley-Terry Modeling :red_circle:

This stage treat the readability scoring task as a text ranking problem, computing Bradley-Terry scores based on the paired excerpts to determine relative difficulty.

### Concatenation :red_circle:

This stage merge the computed ranking scores with the classic linguistic features.

### Final Regression :red_circle:

This stage feed the concatenated feature set into a linear regressor to output the final absolute readability score (sklearn.linear_model).
