## Pipeline Stages

### Client-ID Reverse Engineering :red_circle:

This stage mathematically combine anonymized hardware specs (D-columns) and time deltas to successfully reconstruct unique user IDs without explicit PII (pandas).

### Group K-Fold split :red_circle:

This stage strictly split cross-validation by the generated User IDs and by Time, intentionally mimicking the train/test split to prevent overfitting (sklearn).

### Target Encoding :red_circle:

This stage apply smoothed target encoding to high-cardinality categorical variables using only historical data relative to the current row (category_encoders).

### Feature Aggregation :white_check_mark:

This stage calculate the rolling mean and standard deviation of transaction amounts per User ID over the last 30 days (pandas).

**Bound atom:** `sciona.atoms.time_series_features.rolling_window_features`

### GBDT modeling :red_circle:

This stage train massive XGBoost and CatBoost models on the engineered features, optimizing for AUC.
