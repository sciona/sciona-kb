# Deepfake Detection Challenge (DFDC) - 1st Place (2020) - Pipeline Template

> This template captures the ordered steps used by Deepfake Detection Challenge (DFDC) - 1st Place (2020): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## When to Use This

- The new task has the same broad problem type: Video understanding / Image classification.
- The available data supports the same ordered transformations and intermediate artifacts as the documented winning pipeline.
- Offline batch processing is acceptable, including feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Frame extraction creates the representation needed by Temporal aggregation, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Frame extraction :red_circle:

This stage sample 32 frames uniformly across each video file (OpenCV).

### Face detection :red_circle:

This stage run an MTCNN/RetinaFace detector on extracted frames to find bounding boxes (PyTorch).

### Face alignment & cropping :red_circle:

This stage apply margin expansions and align facial landmarks to a standard template, cropping to just the face (OpenCV/Albumentations).

### Deep fake classification :red_circle:

This stage train an ensemble of EfficientNet-B7 and Se-ResNeXt50 on the cropped faces with heavy image augmentations (PyTorch).

### Temporal aggregation :red_circle:

This stage average the frame-level deepfake probability predictions across the video to produce a single score (numpy).
