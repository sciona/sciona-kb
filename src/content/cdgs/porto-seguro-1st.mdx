# Porto Seguro Safe Driver Prediction 1st Place — Denoising Autoencoder Feature Engineering

> Predict which drivers will file insurance claims by first training a neural network to reconstruct noisy versions of the input data, then using its internal representations as extra features alongside the originals, and combining predictions from several different model types.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.porto_seguro_1st" readOnly={true} /> */}

## When to Use This

- Tabular data with >50 features and high class imbalance (>10:1)
- Missing values encoded as sentinel values (-1, 999, etc.) rather than truly absent
- No strong domain-specific feature engineering available — raw features need nonlinear interaction discovery
- Heterogeneous feature types (mix of categorical and continuous) where manual interaction terms are intractable

## Key Insight

The DAE learns a smooth manifold of the feature space. Its hidden activations capture non-linear feature interactions that tree models cannot discover from raw features alone. The swap-noise corruption forces the DAE to learn robust, denoised representations rather than memorizing individual rows, because swap noise preserves marginal distributions while destroying pairwise correlations.

## Pipeline Stages

### Missing Value Imputation :warning:

Clean up the data by replacing placeholder missing values and converting category columns into numbers.

**Bound atom:** `sciona.atoms.ml.sklearn.impute.simple_imputer_transform`

### Denoising Autoencoder Features :white_check_mark:

Train a neural network to reconstruct the original data from a randomly scrambled version, then use the network's internal representation as new features that capture patterns in the data.

**Bound atom:** `sciona.atoms.dl.architectures.denoising_autoencoder`

### Feature Concatenation :red_circle:

Combine the original data columns with the new features learned by the autoencoder into one wider table.

### Model Training :warning:

Train three different types of prediction models on the enriched feature set: two tree-based models and a neural network.

**Bound atom:** `sciona.atoms.dl.architectures.lightgbm_train`

### Rank Average Ensemble :warning:

Instead of averaging raw predictions, rank all predictions from each model separately, then average the ranks to get a final ordering that is robust to differences in how each model scores confidence.

**Bound atom:** `sciona.atoms.dl.image_augmentation.fold_ensemble_average`
