## Key Insight

The load-bearing idea is the pipeline topology: State spatialization creates the representation needed by Heuristic Override, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### State spatialization :red_circle:

This stage convert the discrete grid-based game state into a massive 3D multi-channel image tensor representing resources, units, and cooldowns (numpy).

### Action space definition :red_circle:

This stage map the complex, multi-unit action space into a flat, predictable array of categorical logits.

### Behavioral Cloning (Imitation Learning) :red_circle:

This stage train a U-Net architecture on the game replays of top-tier bot matches to accurately predict expert moves (PyTorch).

### RL Fine-tuning :red_circle:

This stage initialize a PPO (Proximal Policy Optimization) agent with the U-Net weights and run self-play simulated matches to optimize the policy beyond human baselines (Stable Baselines 3 / Ray RLlib).

### Heuristic Override :red_circle:

This stage intercept RL actions during inference and strictly enforce hard-coded rules for edge cases (e.g., forcing units to build a city if they are about to die at night) (Python).
