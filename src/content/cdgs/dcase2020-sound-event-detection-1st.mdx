## Pipeline Stages

### Audio conversion :red_circle:

This stage generate standard Log-Mel Spectrograms from both labeled and weakly-labeled datasets (librosa).

### Mean-Teacher framework initialization :red_circle:

This stage instantiate two identical CNN-RNN architectures (CRNNs): one designated "Student", one designated "Teacher" (PyTorch).

### Supervised phase :red_circle:

This stage train the Student model normally on the small set of ground-truth labeled audio samples.

### Unsupervised consistency phase :red_circle:

This stage pass unlabeled audio through both models. Use MSE loss to force the Student model's predictions to match the Teacher model's predictions, constantly updating the Teacher via Exponential Moving Average (EMA) of the Student's weights (PyTorch).

### Post-processing :red_circle:

This stage apply a mathematical Median Filter to the frame-level predictions to eliminate impossible rapid clicking/switching between active sound classes (scipy.signal).
