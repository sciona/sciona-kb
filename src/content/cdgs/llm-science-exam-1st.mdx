## Key Insight

The load-bearing idea is the pipeline topology: Offline Indexing creates the representation needed by Confidence weighting, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Offline Indexing :warning:

This stage embed the entire Wikipedia dataset using a sentence-transformer (HuggingFace).

**Bound atom:** `sciona.atoms.dl.embeddings.build_faiss_flat_ip`

### Dense Retrieval :red_circle:

This stage embed the test question and use FAISS to extract the top 5 most relevant Wikipedia paragraphs.

### Context concatenation :red_circle:

This stage construct a prompt string: [Context] + [Question] + [Option A, B, C, D, E].

### LLM inference :red_circle:

This stage process the string through a fine-tuned DeBERTa-V3-Large model equipped with a 5-way multiple choice classification head (PyTorch/HuggingFace).

### Confidence weighting :red_circle:

This stage average the multiple-choice probabilities across 3 different retrieved context permutations to produce final ranking.
