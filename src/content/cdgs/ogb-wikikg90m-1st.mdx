## Key Insight

## Pipeline Stages

### Language feature embedding :red_circle:

This stage process the text descriptions of all 90 million Wikipedia entities using a pre-trained RoBERTa model to output dense 768D text vectors (HuggingFace).

### Relational embedding (TransE) :red_circle:

This stage train a TransE translation model over the Knowledge Graph triples (Head, Relation, Tail) using the text vectors as initialization weights (DGL-KE).

### Relational embedding (ComplEx) :red_circle:

This stage train a ComplEx model in parallel to capture non-symmetric and inverse relation logic across the graph (DGL-KE).

### Candidate generation :red_circle:

This stage perform a fast K-Nearest Neighbor query in the embedded graph space to retrieve the top 100 most likely target entities for a missing link (FAISS).

### MLP Re-ranking :red_circle:

This stage concatenate the embeddings of the retrieved Head, Relation, and candidate Tail, passing them through an MLP to score and re-sort the final 100 choices (PyTorch).
