# MICCAI TN-SCUI - 1st Place (2020) - Pipeline Template

> This template captures the ordered steps used by MICCAI TN-SCUI - 1st Place (2020): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## When to Use This

- The new task has the same broad problem type: Semantic segmentation.

## Key Insight

The load-bearing idea is the pipeline topology: Data Loading creates the representation needed by Binarization, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Data Loading :red_circle:

This stage ingest single-target medical imagery specifically focused on thyroid nodules (PIL, numpy).

### Image Normalization :red_circle:

This stage apply standard intensity normalization and min-max scaling to standardize the dynamic range of the ultrasound imagery.

### Segmentation Training :red_circle:

This stage train an encoder-decoder architecture optimized heavily for single-target, high-contrast boundaries (torch).

### TTA Inference :red_circle:

This stage apply deterministic flips and rotations during prediction to smooth out spatial variance.

### Binarization :red_circle:

This stage threshold the continuous probability maps into strict binary output masks.
