## Key Insight

The load-bearing idea is the pipeline topology: Data leak extraction creates the representation needed by Rank Averaging, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Data leak extraction :red_circle:

This stage algorithmically trace document IDs between page views and ad clicks, generating a strict deterministic matching feature that bypasses ML completely (pandas).

### Feature hashing :white_check_mark:

This stage hash billions of sparse categorical features (user IDs, geo-locations, ad IDs) into a fixed-width vector space to fit into memory (Vowpal Wabbit).

**Bound atom:** `sciona.atoms.ml.sklearn.feature_extraction.feature_hasher_shell.feature_hasher_csr_matrix`

### Field-aware Factorization Machine (FFM) :red_circle:

This stage train an FFM architecture heavily optimized for sparse click-through-rate predictions (libffm).

### Follow-The-Regularized-Leader (FTRL) :red_circle:

This stage train an online FTRL model passing linearly through the dataset row-by-row, constantly updating its weights to handle chronological concept drift (Vowpal Wabbit).

### Rank Averaging :red_circle:

This stage sort the outputs of the FFM and FTRL models per user, and average their sorted ranks to generate final ad placements (scipy).
