## Key Insight

The load-bearing idea is the pipeline topology: Vector rasterization creates the representation needed by Vectorization, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Vector rasterization :red_circle:

This stage convert the provided WKT polygon strings into dense binary masks mapping perfectly to the GeoTIFF coordinates (OpenCV/GDAL).

### Multi-resolution concatenation :red_circle:

This stage upsample the 16-band multispectral low-resolution images to match the high-resolution panchromatic images, concatenating them into a unified tensor (scipy).

### U-Net Segmentation :red_circle:

This stage train an ensemble of deep U-Nets to predict probability masks for 10 distinct terrain classes (Keras).

### Conditional Random Fields (CRF) :white_check_mark:

This stage apply dense CRFs to the output masks to strictly snap the blurry segmentation predictions to the sharp visible edges of the original image (pydensecrf).

**Bound atom:** `sciona.atoms.dl.segmentation.dense_crf_2d`

### Vectorization :red_circle:

This stage execute contour finding algorithms to trace the CRF masks, applying Douglas-Peucker simplification to output clean multi-polygons (Shapely/OpenCV).
