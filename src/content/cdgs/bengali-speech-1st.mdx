## Key Insight

The load-bearing idea is the pipeline topology: Audio normalization creates the representation needed by Decoding, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Audio normalization :red_circle:

This stage resample all audio files to 16kHz and normalize volume levels (librosa/torchaudio).

### Feature extraction :red_circle:

This stage pass raw audio through the Whisper encoder/decoder architecture to extract logits (HuggingFace).

### Model fine-tuning :red_circle:

This stage fine-tune the Whisper-Medium architecture using Connectionist Temporal Classification (CTC) loss.

### Language modeling :red_circle:

This stage construct a KenLM 5-gram language model based on the Bengali Wikipedia corpus.

### Decoding :red_circle:

This stage apply beam search decoding using the KenLM language model to correct phonetically similar Bengali spelling mistakes (pyctcdecode).
