## Key Insight

The load-bearing idea is the pipeline topology: Top-bot scraping creates the representation needed by Inference compiling, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Top-bot scraping :red_circle:

This stage constantly scrape replays of the top 50 bots off the competition leaderboard to serve as dynamic training data (requests).

### Value Network training :red_circle:

This stage train a ResNet architecture to input the Halite board state and output the final expected win probability of the game (PyTorch).

### Policy Network training :red_circle:

This stage train a separate architecture to predict the exact next move of the expert bots (Behavioral Cloning).

### Monte Carlo Tree Search (MCTS) :red_circle:

This stage implement a highly optimized MCTS in a low-level language. Use the Policy network to guide the search tree and the Value network to evaluate terminal states (custom C++).

### Inference compiling :red_circle:

This stage compile the C++ MCTS algorithm into a .so file executable by the Python competition environment.
