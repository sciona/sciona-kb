# Santander Customer Transaction Prediction 1st Place — Synthetic Data Detection with Count Encoding

> Predict which bank customers will make a transaction by first figuring out which test-set rows are fake padding (added by the competition organizers), removing them so they do not corrupt frequency statistics, then counting how often each value appears and using those counts as features for a gradient-boosted tree model.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.santander_transaction_1st" readOnly={true} /> */}

## When to Use This

- Tabular binary classification with many anonymized numeric features where domain-specific feature engineering is impossible
- Test set may contain synthetic or padded rows that corrupt frequency-based statistics if not removed
- Features have discrete or semi-discrete value distributions where frequency counts carry discriminative signal

## Key Insight

The real insight is detecting and removing synthetic padding rows from the test set. The competition organizers injected fake rows with unique values to prevent leakage-based solutions, but this injection itself leaks information: real rows share values with the training set while fake rows have values appearing exactly once. Removing fake rows before computing frequency counts ensures the count features reflect the true data distribution, and these clean counts become the dominant signal for the downstream classifier.

## Pipeline Stages

### Count Encoding :white_check_mark:

For each feature column, count how many times each value appears in the data.

**Bound atom:** `sciona.atoms.ml.tabular.gradient_boosting.frequency_encode`

### Fake Data Identification :red_circle:

Find test-set rows that have values appearing nowhere else in the data — these are artificial rows added as padding and should not be used.

### Real Data Filtering :red_circle:

Remove the fake rows from the test set so they do not pollute the frequency counts.

### Value Substitution :warning:

Swap each feature value for how often it appears in the data, and also keep the original values alongside.

**Bound atom:** `sciona.atoms.ml.tabular.gradient_boosting.frequency_encode`

### LightGBM Training :white_check_mark:

Train a gradient-boosted tree model using only a random subset of features and data rows at each step to prevent overfitting.

**Bound atom:** `sciona.atoms.dl.architectures.lightgbm_train`

### Shuffle Augmentation :red_circle:

Create extra training examples by randomly rearranging feature values among rows with the same label, giving the model more data to learn from.
