## Key Insight

The load-bearing idea is the pipeline topology: Stain Translation creates the representation needed by Contour smoothing, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Stain Translation :red_circle:

This stage use CycleGAN to strictly translate the color profiles of the HPA dataset (immunofluorescence) to perfectly match the color profiles of the HuBMAP dataset (PAS stain) (PyTorch).

### Pseudo-labeling :white_check_mark:

This stage use models trained on HuBMAP to generate masks for the translated HPA images, creating a massive new supervised dataset.

**Bound atom:** `sciona.atoms.ml.sklearn.semi_supervised.self_training_select_pseudo_labels`

### Multi-organ Backbone :red_circle:

This stage train a U-Net / CoatNet architecture to segment all 5 organs simultaneously, rather than training 5 separate models (segmentation_models_pytorch).

### Organ-specific thresholds :red_circle:

This stage run an optimization loop to find the perfect pixel-probability cutoff threshold independently for each organ type (e.g., lungs vs kidneys) (scipy).

### Contour smoothing :red_circle:

This stage apply morphological closing and small-hole filling to the binary masks to ensure biologically coherent outputs (OpenCV).
