# Image Matching Challenge 1st Place — Fundamental Matrix from Local Feature Matching (2022)

> Find matching points between two photos using multiple neural-network detectors, combine all the matches, then use a robust algorithm to throw out wrong matches and compute the geometric relationship between the two camera views.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.image_matching_1st" readOnly={true} /> */}

## When to Use This

- Image pairs contain a mix of textured and textureless regions, so no single matcher type can achieve full coverage
- Camera geometry recovery (fundamental/essential matrix) is the target, not just correspondence finding
- Images may have extreme aspect ratios or resolution variation that require dynamic resizing before feature extraction
- High outlier rates are expected in matches, requiring a robust estimator like MAGSAC++ rather than vanilla RANSAC

## Key Insight

Using BOTH sparse (SuperPoint/SuperGlue) and dense (LoFTR) matchers in parallel, then concatenating their correspondences before MAGSAC++, covers both textured and textureless regions. Neither matcher alone achieves full coverage: sparse methods miss low-texture areas, dense methods produce noisier matches in high-texture areas. The combined pool gives MAGSAC++ a richer inlier set to work with.

## Pipeline Stages

### Dynamic Resizing :red_circle:

Resize unusually shaped images (very tall or very wide) so that feature detection networks can process them reliably.

### SuperPoint Extraction :white_check_mark:

Use a neural network to find distinctive points in each image and describe each point with a numerical fingerprint for later matching.

**Bound atom:** `sciona.atoms.dl.architectures.superpoint_extractor`

### LoFTR Extraction :white_check_mark:

Use a different neural network that directly finds matching points between two images at once, producing many more matches than the sparse detector.

**Bound atom:** `sciona.atoms.dl.architectures.loftr_dense_matcher`

### SuperGlue Matching :white_check_mark:

Use a neural network to figure out which distinctive points in image 1 correspond to which distinctive points in image 2, using the numerical fingerprints from SuperPoint.

**Bound atom:** `sciona.atoms.dl.architectures.superglue_matcher`

### Match Concatenation :red_circle:

Merge all the matched point pairs from both methods into one big list so the geometry estimator can work with all of them together.

### Fundamental Matrix Estimation :warning:

Use a robust algorithm to figure out which matched points are correct and which are wrong, then compute the geometric relationship (fundamental matrix) between the two camera views from only the correct matches.

**Bound atom:** `sciona.atoms.dl.perception_3d.ransac_homography`
