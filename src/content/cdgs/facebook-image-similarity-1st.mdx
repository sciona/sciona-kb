# Facebook AI Image Similarity Challenge - 1st Place (2021) - Pipeline Template

> This template captures the ordered steps used by Facebook AI Image Similarity Challenge - 1st Place (2021): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## When to Use This

- The new task has the same broad problem type: Metric Learning / Image Retrieval.
- The available data supports the same ordered transformations and intermediate artifacts as the documented winning pipeline.
- Offline processing is acceptable, including search, feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Contrastive Augmentation creates the representation needed by Spatial Verification, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Contrastive Augmentation :red_circle:

This stage heavily apply cropping, color jitter, blur, and overlaid text/emojis during training to mimic highly manipulated social media images (Albumentations).

### Backbone architecture :red_circle:

This stage train a massive EfficientNet-V2 / Swin Transformer architecture using ArcFace loss and GeM pooling to output robust 256D representations (PyTorch).

### FAISS Indexing :red_circle:

This stage project all 1 million reference images into the 256D embedding space and load them into an exact L2 distance FAISS index.

### Database-Side Augmentation (Query Expansion) :white_check_mark:

This stage query the database using the test image; extract the top 3 results, average their embeddings with the test image embedding, and query the database a second time to pull deeply hidden matches (numpy).

**Bound atom:** `sciona.atoms.dl.embeddings.alpha_query_expansion`

### Spatial Verification :warning:

This stage run an exact geometry check on the top returned pairs to confirm actual overlap, reranking the final output (OpenCV).

**Bound atom:** `sciona.atoms.ml.sklearn.linear_model.ransac.ransac_consensus_is_better`
