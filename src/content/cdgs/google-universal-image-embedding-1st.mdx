# Google Universal Image Embedding - 1st Place (2022) - Pipeline Template

> This template captures the ordered steps used by Google Universal Image Embedding - 1st Place (2022): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## When to Use This

- The new task has the same broad problem type: Representation Learning / Metric Learning.

- Offline processing is acceptable, including search, feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Image resizing creates the representation needed by L2 Normalization, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Image resizing :red_circle:

This stage dynamically resize diverse multi-domain images (landmarks, apparel, artwork) to fixed dimensions, applying heavy domain-agnostic augmentations (Albumentations).

### Transformer Backbone :red_circle:

This stage pass images through a massive ViT-Huge architecture to extract high-dimensional representation features (timm/PyTorch).

### Sub-center ArcFace :red_circle:

This stage train the model using a Sub-center ArcFace loss function, which creates multiple cluster centers per class to handle visually dissimilar items that belong to the same category (e.g., front vs back of a dress) (PyTorch).

### PCA dimensionality reduction :white_check_mark:

This stage fit a Principal Component Analysis (PCA) model to compress the resulting 1024D embeddings strictly down to 64D as required by the competition constraints (sklearn).

**Bound atom:** `sciona.atoms.dl.embeddings.pca_whiten_reduce`

### L2 Normalization :red_circle:

This stage normalize the final 64D vector to project all embeddings onto a hypersphere for perfect cosine similarity retrieval (numpy).
