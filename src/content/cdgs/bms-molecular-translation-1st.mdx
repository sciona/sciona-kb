## Key Insight

The load-bearing idea is the pipeline topology: Image preprocessing creates the representation needed by Ensembling, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Image preprocessing :red_circle:

This stage invert colors, pad images to specific aspect ratios, and apply rotation augmentations (OpenCV/Albumentations).

### CNN Encoder :red_circle:

This stage pass the image through a CNN backbone (EfficientNet/Swin Transformer) to extract a 2D spatial feature grid (PyTorch).

### Transformer Decoder :red_circle:

This stage process the spatial features using an autoregressive Transformer decoder predicting InChI sequence tokens one by one (PyTorch).

### Decoding :white_check_mark:

This stage use Beam Search (width 5) to generate the most probable chemical string sequences (custom script).

**Bound atom:** `sciona.atoms.ml.text_nlp.beam_search`

### Ensembling :red_circle:

This stage average sequence probabilities at each decoding step across multiple model checkpoints.
