## Pipeline Stages

### Data Deduplication :red_circle:

This stage clean the input diagnostic math multiple-choice questions and permanently drop duplicate text rows to prevent data leakage (pandas).

### Prefix Formatting :red_circle:

This stage organize text inputs into a highly structured "prefix-shared" format, combining the overarching context, the correct answer, and multiple candidate misconceptions into a single continuous prompt block.

### Attention Masking :white_check_mark:

This stage utilize custom attention masks (FlexAttention) to explicitly map the causal relationships between the shared prefix and the multiple suffix candidates without cross-contamination (torch.nn.attention).

**Bound atom:** `sciona.atoms.dl.architectures.flex_attention_mask`

### Feature Extraction :red_circle:

This stage extract the isolated last-token features from the concatenated sequences.

### SFT Head :red_circle:

This stage feed the extracted features into a standard linear classifier (nn.Linear) optimized strictly with Cross-Entropy Loss to initially score the candidates via Supervised Fine-Tuning.

### Reasoning Optimization :red_circle:

This stage fine-tune a secondary, deeper checkpoint utilizing Group Relative Policy Optimization (GRPO) on significantly longer context windows to capture extended logic.

### Model Merging :red_circle:

This stage merge the distinct SFT and GRPO weights linearly to strike an optimal balance between token efficiency and complex reasoning accuracy (peft, torch).
