# NeurIPS Open Polymer Prediction - 1st Place (2025) - Pipeline Template

> This template captures the ordered steps used by NeurIPS Open Polymer Prediction - 1st Place (2025): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## Key Insight

The load-bearing idea is the pipeline topology: Chemical Descriptor Extraction creates the representation needed by Exact Match Override, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Chemical Descriptor Extraction :red_circle:

This stage process SMILES chemical strings to structurally generate RDKit molecular descriptors and Morgan fingerprints (rdkit).

### Data Augmentation :red_circle:

This stage apply randomized SMILES generation protocols (10x augmentation) to create robust text representations of identical molecules.

### LM Representation :red_circle:

This stage fine-tune a code-pretrained language model (CodeBERTa) on the SMILES strings to predict multiple continuous properties simultaneously (transformers, torch).

### Tabular Modeling :red_circle:

This stage parallelly train an AutoGluon tabular model specifically on the generated RDKit/Morgan descriptor outputs (autogluon).

### Ensembling :red_circle:

This stage calculate target-specific weighted averages across both streams (heavily weighting the BERT model, assigning it a 0.91-0.97 coefficient).

### Distribution Shift Post-processing :red_circle:

This stage shift the predictions for specific targets (e.g., Tg) by statistical factors (std * 0.5644) to manually correct a known train/test distribution mismatch (numpy).

### Exact Match Override :red_circle:

This stage intercept the pipeline output and forcefully replace predictions with ground-truth values if the test SMILES perfectly matches a known training sample.
