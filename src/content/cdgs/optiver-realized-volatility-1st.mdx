## Pipeline Stages

### LOB parsing :white_check_mark:

This stage compute the Weighted Average Price (WAP) and log returns continuously across the raw bid/ask tick data (pandas).

**Bound atom:** `sciona.atoms.financial_signals.weighted_average_price`

### Time-bucketing :red_circle:

This stage aggregate the tick-level data into fixed historical time windows (e.g., last 600 seconds, 300 seconds, 100 seconds) computing realized volatility per bucket (numpy).

### Feature clustering :white_check_mark:

This stage cluster different financial stocks based on their historical volatility correlations to create "market sector" proxy features (scikit-learn KMeans).

**Bound atom:** `sciona.atoms.ml.sklearn.cluster.kmeans_plusplus.kmeans_plusplus_initialize_dense`

### Cross-sectional aggregation :white_check_mark:

This stage compute the mean volatility of the entire market at each specific time bucket to contextualize individual stock movements (pandas).

**Bound atom:** `sciona.atoms.ml.tabular.gradient_boosting.group_aggregate`

### Custom Objective GBDT :red_circle:

This stage train LightGBM models utilizing a custom Root Mean Square Percentage Error (RMSPE) loss function directly aligning with the competition metric (LightGBM).
