## Key Insight

The load-bearing idea is the pipeline topology: Frame extraction creates the representation needed by Non-Maximum Suppression (NMS), so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Frame extraction :red_circle:

This stage sample videos at 25 FPS and resize to 480p (OpenCV).

### 2.5D CNN extraction :red_circle:

This stage stack 3 consecutive video frames into the RGB channels of a single image and pass through an EfficientNet to extract spatial features (PyTorch).

### SlowFast extraction :red_circle:

This stage run a SlowFast 3D-CNN directly on a window of video frames to extract temporal features (PyTorchVideo).

### Sequence modeling :red_circle:

This stage concatenate spatial and temporal features per frame, and pass the sequence through a 1D CNN / Transformer layer (PyTorch).

### Non-Maximum Suppression (NMS) :red_circle:

This stage locate peaks in the 1D probability output to pinpoint the exact frame of the action (pass, foul, shot) (scipy).
