## Key Insight

## Pipeline Stages

### Time aggregation :red_circle:

This stage aggregate the raw daily transaction logs explicitly into monthly shop-item groupings (pandas).

### Target clipping :red_circle:

This stage strictly clip the target variable between [0, 20] to mathematically bound the loss function and replicate the competition's exact evaluation metric (numpy).

### Lag feature generation :red_circle:

This stage generate lag columns (shifting historical sales values by 1, 2, 3, 6, and 12 months) and append them as columns to the current row (pandas).

### Mean encoding :white_check_mark:

This stage calculate the expanding mean of sales per shop and per item over time to generate robust continuous features.

**Bound atom:** `sciona.atoms.ml.tabular.gradient_boosting.target_encode`

### GBDT modeling :red_circle:

This stage train a standard XGBoost or LightGBM model utilizing RMSE loss on the engineered row data.
