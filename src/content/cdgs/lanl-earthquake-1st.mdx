## Key Insight

The load-bearing idea is the pipeline topology: Signal chunking creates the representation needed by Stacking, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Signal chunking :white_check_mark:

This stage slice the continuous 1D acoustic data into fixed-size chunks of 150,000 readings (numpy).

**Bound atom:** `sciona.atoms.dl.video_temporal.sliding_windows`

### Time-domain features :red_circle:

This stage calculate mean, standard deviation, quantiles, and rolling peak counts per chunk (pandas).

### Frequency-domain features :red_circle:

This stage apply Welch's method to extract dominant frequency bands from the acoustic chunks (scipy.signal).

### Adversarial validation :red_circle:

This stage train a classifier to distinguish train vs test data, and drop features that drastically differ between sets (LightGBM).

### Model training :red_circle:

This stage fit Support Vector Regressors (SVR), Kernel Ridge Regression, and LightGBM models.

### Stacking :red_circle:

This stage average the outputs and apply a strict floor/ceiling based on physical bounds of the earthquake cycle.
