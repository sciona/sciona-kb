## Key Insight

The load-bearing idea is the pipeline topology: Language merging creates the representation needed by Ensembling, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Language merging :red_circle:

This stage translate external Hindi/Tamil QA datasets to English, and English datasets (SQuAD) to Hindi/Tamil for heavy cross-lingual pretraining (Google Translate API).

### Tokenization :white_check_mark:

This stage tokenize question and context strings using Muril, XLM-RoBERTa, and RemBERT tokenizers (HuggingFace).

**Bound atom:** `sciona.atoms.ml.tokenizer.tokenize`

### Span modeling :red_circle:

This stage train models using a standard Question-Answering head (two linear layers predicting start_token_logits and end_token_logits) (PyTorch).

### Post-processing logic :warning:

This stage iterate through the top 20 start/end token predictions, penalize answers that are too long (e.g., > 100 characters), and select the highest scoring valid span.

**Bound atom:** `sciona.atoms.ml.text_nlp.char_to_token_offsets`

### Ensembling :red_circle:

This stage average the start and end logits across 5 different model architectures before executing the post-processing.
