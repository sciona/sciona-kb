# OTTO - Multi-Objective Recommender System - 1st Place (2023) - Pipeline Template

> This template captures the ordered steps used by OTTO - Multi-Objective Recommender System - 1st Place (2023): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## When to Use This

- The new task has the same broad problem type: Learning-to-rank / Session-based RecSys.

- Offline batch processing is acceptable, including feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Graph construction creates the representation needed by Post-processing, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Graph construction :red_circle:

This stage build item-to-item co-visitation matrices based on historical user sessions (pandas/cuDF).

### Candidate generation :red_circle:

This stage use the co-visitation graph to retrieve the top 100 likely next items for a given user session (RAPIDS cuDF).

### Feature engineering :red_circle:

This stage calculate user stats, item popularity, and user-item interaction frequency from the candidate pairs (pandas).

### Ranker model :red_circle:

This stage train an XGBoost LambdaMART model on the candidates to predict clicks, carts, and orders.

### Post-processing :red_circle:

This stage sort candidates by predicted score and truncate to the top 20 recommendations per user.
