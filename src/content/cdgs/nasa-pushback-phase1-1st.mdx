## Key Insight

The load-bearing idea is the pipeline topology: Data Ingestion creates the representation needed by Concatenation, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Data Ingestion :red_circle:

This stage load public air traffic and terminal weather data into tabular dataframes (pandas).

### Entity Splitting :red_circle:

This stage split the dataset categorically by individual airport rather than treating the data as a single monolithic block.

### Parallel Modeling :red_circle:

This stage train an independent CatBoost regressor sequence dedicated specifically to each individual airport's telemetry (catboost).

### Airline Sequencing :red_circle:

This stage within each airport model, train sub-models in strict sequence across the different operating airlines to capture sequential delays.

### Concatenation :red_circle:

This stage re-concatenate the distinct airport and airline predictions into a single standardized submission file.
