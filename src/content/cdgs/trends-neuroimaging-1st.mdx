## Key Insight

The load-bearing idea is the pipeline topology: Brain mask application creates the representation needed by Multi-head modeling, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Brain mask application :red_circle:

This stage map functional MRI matrices back into their 3D spatial voxel coordinates (numpy/nibabel).

### Independent Component Analysis :red_circle:

This stage calculate cross-correlations (FNC) between brain spatial maps across subjects.

### Ridge feature engineering :red_circle:

This stage run thousands of individual Ridge regressions treating every variable pair as dependent/independent to extract coefficients as metaparameters (sklearn).

### Feature concatenation :red_circle:

This stage flatten raw FNC maps, ICA loadings, and Ridge coefficients into a 1D tabular vector per patient.

### Multi-head modeling :red_circle:

This stage train separate Support Vector Regressors and Nu-SVR models for each of the 5 target variables (sklearn).
