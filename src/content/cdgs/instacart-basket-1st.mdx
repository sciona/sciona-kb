# Instacart Market Basket Analysis 1st Place — F1-Optimal Reorder Set Prediction

> Predict which grocery products a customer will reorder by first estimating the probability of each product being reordered using gradient boosting, then using a clever algorithm to pick the exact set of products that maximizes the F1 scoring metric.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.instacart_basket_1st" readOnly={true} /> */}

## When to Use This

- Task requires predicting a variable-size set of items per entity (e.g., products per user, tags per document) and evaluation metric is F1
- Per-item probabilities can be estimated independently but the optimal subset depends on their joint distribution under the F1 metric
- Rich historical interaction data is available for feature engineering (repeat purchase patterns, temporal recency, frequency)

## Key Insight

Faron's F1 maximization DP is the critical innovation. Given per-item reorder probabilities from any classifier, naive thresholding (select all items with p > 0.5) is suboptimal for F1 because the optimal threshold depends on the number of true positives, which is unknown. The DP algorithm exactly computes the expected F1 for every possible subset size and selects the subset that maximizes it, turning an intractable combinatorial problem into an O(n^2) computation under the independence assumption.

### User-Product Aggregation :red_circle:

### Candidate Generation :red_circle:

### Feature Merging :red_circle:

### Reorder Probability Model :white_check_mark:
