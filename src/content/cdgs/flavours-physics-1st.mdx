# Flavours of Physics 1st Place -- Constraint-Satisfying Classification

> Train three separate classifiers with different strategies for avoiding bias toward a protected variable (particle mass): one uses only safe features, one penalizes mass-correlated predictions during training, and one adds random noise afterward. Each must pass statistical tests proving its predictions are independent of mass before being accepted.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.flavours_physics_1st" readOnly={true} /> */}

## When to Use This

- Classification where predictions must be statistically independent of a protected/sensitive variable (fairness, physics mass independence)
- Hard constraint gates (not soft regularization) must be satisfied before model quality is evaluated
- Multiple decorrelation strategies should be explored in parallel because the accuracy-fairness tradeoff is problem-specific
- Tabular data where gradient boosting is the natural model family and decorrelation must be integrated into the training loop

## Key Insight

The flatness penalty gradient pushes per-bin CDFs toward the global CDF, enforcing statistical independence between predictions and the protected variable during training. This is a training-time penalty (not post-hoc), so the model learns representations that are intrinsically decorrelated from the protected variable rather than having correlation removed after the fact.

## Pipeline Stages

### Safe Feature Selection :red_circle:

Pick only the features that are guaranteed not to leak information about the protected variable (mass), even though this limits the model's power.

### Flatness-Constrained Gradient Boosting :white_check_mark:

Train a tree-based classifier that is explicitly penalized whenever its predictions become correlated with the protected variable, learning to ignore mass-related patterns during training itself.

**Bound atom:** `sciona.atoms.ml.constrained_ml.decorrelation.flatness_penalty_gradient`

### Noise Injection Decorrelation :white_check_mark:

Add random noise to a strong model's predictions to wash out any correlation with the protected variable, trading some accuracy for constraint compliance.

**Bound atom:** `sciona.atoms.ml.constrained_ml.decorrelation.noise_injection_decorrelation`

### KS Agreement Test :white_check_mark:

Check whether the model gives similar predictions on real data and simulated data -- if not, the model may be exploiting simulation artifacts rather than real physics.

**Bound atom:** `sciona.atoms.ml.constrained_ml.decorrelation.compute_ks_agreement`

### CvM Mass Decorrelation Test :white_check_mark:

Check whether the model's predictions change depending on particle mass -- if they do, the model is biased and must be rejected. Uses a statistical test that is insensitive to how predictions are scaled.

**Bound atom:** `sciona.atoms.ml.constrained_ml.decorrelation.compute_cvm_mass_decorrelation`

### Truncated Weighted ROC AUC :white_check_mark:

Measure classifier quality focusing only on the high-confidence predictions, ignoring performance on hard-to-classify examples where the model guesses.

**Bound atom:** `sciona.atoms.ml.constrained_ml.decorrelation.roc_auc_truncated_weighted`
