# Jigsaw Unintended Bias in Toxicity Classification 1st Place — Bias-Aware Transformer Ensemble

> Detect toxic comments while being fair across demographic groups by training multiple large language models with a custom loss function that pays extra attention to comments mentioning specific identities, applying a trick with multiple dropout masks to stabilize predictions, and averaging the results.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.jigsaw_toxicity_bias_1st" readOnly={true} /> */}

## When to Use This

- Text classification task where evaluation explicitly penalizes bias against demographic subgroups (fairness-aware metric)
- Training labels contain annotation bias — annotators systematically rate identity-mentioning text as more toxic regardless of actual toxicity
- Multiple pre-trained transformer architectures are available and diversity in tokenization/pre-training improves ensemble robustness

## Key Insight

Demographic-weighted BCE explicitly prevents the model from learning that certain identity terms correlate with toxicity labels due to annotation bias. Without this correction, the model learns a spurious shortcut: 'comment mentions a demographic group' -> 'probably toxic,' because annotators disproportionately labeled identity-mentioning comments as toxic. The weighted loss upweights identity-mentioning samples during training, forcing the model to find the actual toxic signal rather than relying on the identity-toxicity correlation baked into the labels.
