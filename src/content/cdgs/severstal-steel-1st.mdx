# Severstal Steel Defect Detection 1st Place — Classifier-Gated Segmentation with Area Thresholding

> Detect steel defects by first using a classifier to decide if an image has any defect at all, then applying detailed segmentation models only to images likely containing defects, and finally removing tiny predicted regions that are probably noise.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.severstal_steel_1st" readOnly={true} /> */}

## When to Use This

- Semantic segmentation where the majority of images (>70%) contain no target objects — high negative rate
- Multi-class pixel-level defect detection on high-resolution industrial images requiring tiling
- False positive reduction is as important as recall — downstream decisions (e.g., production line stops) have high cost
- Defect regions are small relative to total image area, making area-based postprocessing effective

## Key Insight

The classifier gate exploits a base-rate asymmetry: when most images are defect-free, segmentation models produce false positives on surface textures and noise that are hard to suppress with pixel-level thresholds alone. By routing known-negative images to an empty mask, the pipeline converts a pixel-level precision problem into a much easier image-level binary classification problem. The area thresholding then handles the residual false positives on positive images, which are predominantly small isolated clusters that real defects never produce.

## Pipeline Stages

### Tile Preprocessing :warning:

Cut large steel images into smaller overlapping patches and apply random flips to create more training examples that fit in GPU memory.

**Bound atom:** `sciona.atoms.dl.image_augmentation.resize_and_pad_apply`

### Classifier Filtering :warning:

Train a simple yes/no classifier to quickly decide if a steel image has any defect at all. Clean images get an empty output immediately without running the slower segmentation models.

**Bound atom:** `sciona.atoms.dl.architectures.efficientnet_backbone`

### Segmentation Models :warning:

Train two types of pixel-level prediction models, both using EfficientNet as a feature extractor, only on images known to contain defects.

**Bound atom:** `sciona.atoms.dl.architectures.unet_2d_segmentation`

### Multistage Inference :red_circle:

A two-step decision process: first check if there is any defect, and only if yes, run the detailed segmentation to find exactly where the defects are.

### Postprocessing :white_check_mark:

Clean up the predictions by removing tiny predicted defect regions that are almost certainly noise, and converting probability scores into final yes/no defect masks.

**Bound atom:** `sciona.atoms.dl.segmentation.filter_components_by_area`
