# Abstraction and Reasoning Challenge (ARC) - 1st Place (2020) - Pipeline Template

> This template captures the ordered steps used by Abstraction and Reasoning Challenge (ARC) - 1st Place (2020): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

- The new task has the same broad problem type: Program Synthesis / Cellular Automata.
- The available data supports the same ordered transformations and intermediate artifacts as the documented winning pipeline.
- Offline processing is acceptable, including search, feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Grid translation creates the representation needed by Inference, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Grid translation :red_circle:

This stage convert the 2D visual color grids into discrete graph representations (connected components, bounding boxes, internal shapes) (custom Python).

### DSL Construction :red_circle:

This stage define a Domain Specific Language (DSL) consisting of standard functional primitives (e.g., rotate, fill, translate, intersect).

### Tree Search :red_circle:

This stage implement an A* search algorithm to iteratively combine DSL primitives into candidate functional programs.

### Program Validation :red_circle:

This stage execute the candidate programs against the provided "train" input grids; if the output matches the "train" output grids perfectly, save the program.

### Inference :red_circle:

This stage apply the successfully discovered functional program to the unseen "test" input grid to generate the final prediction (Python).
