## Key Insight

The load-bearing idea is the pipeline topology: Synthetic data generation creates the representation needed by Selection, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Synthetic data generation :red_circle:

This stage use local LLMs (Mistral) to generate thousands of "original text -> rewritten text -> prompt used" triplets (vLLM).

### Embedding extraction :warning:

This stage pass the "original text" and the "rewritten text" through a high-quality sentence-transformer (e.g., MiniLM / BGE) to extract dense vectors (HuggingFace).

**Bound atom:** `sciona.atoms.dl.architectures.sentence_transformer`

### Delta vector calculation :red_circle:

This stage subtract the original vector from the rewritten vector to isolate the mathematical "direction" of the prompt transformation (numpy).

### Prompt retrieval :red_circle:

This stage embed a massive database of thousands of known prompts, and calculate the Cosine Similarity between the "delta vector" and the known prompts (FAISS/sklearn).

### Selection :red_circle:

This stage return the string of the prompt with the highest cosine similarity to the vector delta.
