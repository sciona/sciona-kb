## Key Insight

The load-bearing idea is the pipeline topology: Audio conversion creates the representation needed by Inference chunking, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Audio conversion :red_circle:

This stage convert 5-minute audio clips into log-mel spectrogram images (librosa).

### Background noise augmentation :red_circle:

This stage overlay pink noise and external environment sounds during training (audiomentations).

### Sound Event Detection (SED) model :red_circle:

This stage train a ResNeSt-50 backbone topped with an attention mechanism to pool frame-level predictions into a clip-level prediction (PyTorch).

### Thresholding optimization :red_circle:

This stage learn a dynamic probability threshold per bird species using validation data.

### Inference chunking :red_circle:

This stage slice test audio into 5-second overlapping windows, predict via SED, and apply custom thresholds (numpy).
