## Key Insight

The load-bearing idea is the pipeline topology: Vocal isolation creates the representation needed by Post-processing, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Vocal isolation :red_circle:

This stage pass raw audio through HTDemucs to strictly isolate the vocal track from the instrumental background (Demucs/torchaudio).

### Phoneme extraction :red_circle:

This stage pass the isolated vocals through an ASR Acoustic Model (Wav2Vec2) to extract frame-level phoneme probability matrices (HuggingFace).

### Text processing :white_check_mark:

This stage translate the provided raw lyric text into phonetic strings using an external grapheme-to-phoneme dictionary (g2p_en).

**Bound atom:** `sciona.atoms.ml.text_nlp.rule_based_g2p`

### Forced Alignment :red_circle:

This stage apply forced Dynamic Time Warping (DTW) to strictly align the predicted frame-level phoneme probabilities with the true textual phoneme sequence (torchaudio / custom DTW).

### Post-processing :red_circle:

This stage smooth the alignment boundaries using a median filter and assign exact start/end timestamps to every word.
