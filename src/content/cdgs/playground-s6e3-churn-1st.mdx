# Kaggle Playground S6E3: Customer Churn - 1st Place (2026) - Pipeline Template

## Key Insight

The load-bearing idea is the pipeline topology: Data Split creates the representation needed by Meta-Stacking, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Data Split :red_circle:

This stage configure a strict 5-fold Stratified K-Fold setup utilizing a fixed seed to ensure absolute parity across the subsequent 150 models (sklearn.model_selection).

### GPU Feature Engineering :red_circle:

This stage generate anchor target encodings, digit extraction features, radix interactions, and apply TF-IDF transformations specifically on numerical features utilizing GPU dataframes (cuml, rapids).

### Nearest Neighbor Extraction :warning:

This stage perform high-speed KDTree lookups to generate proximity-based tabular features, effectively translating spatial relationships into tabular columns (scipy.spatial).

**Bound atom:** `sciona.atoms.ml.sklearn.neighbors.nearest_neighbors_fit`

### Model Zoo Generation :red_circle:

This stage train a massive ensemble of 150 diverse models spanning 4 GBDT libraries (XGBoost, LightGBM, CatBoost, HistGradientBoosting) and 25 deep learning architectures (Transformers, Graph networks, Multiplicative modules) automatically tuned via Optuna.

### Greedy Forward Selection :red_circle:

This stage execute a hill-climbing algorithm on the Out-Of-Fold (OOF) AUC metrics to iteratively greedily select the highest-contributing models for the final ensemble (numpy, sklearn.metrics).

### Meta-Stacking :red_circle:

This stage train a final Logistic Regression stacker model to learn the specific regional trust weights across the selected ensemble, allowing the system to depend on different sub-models in different regions of the feature space (sklearn.linear_model).
