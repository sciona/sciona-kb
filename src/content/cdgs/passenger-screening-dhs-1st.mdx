## Key Insight

The load-bearing idea is the pipeline topology: Format conversion creates the representation needed by Max Pooling Ensembling, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Format conversion :red_circle:

This stage convert proprietary .a3d mmWave scanner files into dense 3D numpy arrays representing physical volumes (numpy).

### Multi-view 2D projection :red_circle:

This stage mathematically project the 3D volumes into 16 distinct 2D angle views (like an X-ray rotating around the body) (scipy/numpy).

### Body zoning :red_circle:

This stage segment the 2D projections into 17 specific anatomical zones (e.g., right bicep, left ankle) using a custom heuristic (Python).

### CNN Feature Extraction :red_circle:

This stage train ResNet architectures on the specific body zones to predict the presence of concealed threats (PyTorch).

### Max Pooling Ensembling :red_circle:

This stage aggregate the predictions across all 16 rotational views by simply taking the maximum probability for each anatomical zone.
