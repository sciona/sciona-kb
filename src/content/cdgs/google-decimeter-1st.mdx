# Google Smartphone Decimeter Challenge - 1st Place (2021) - Pipeline Template

> This template captures the ordered steps used by Google Smartphone Decimeter Challenge - 1st Place (2021): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## Key Insight

The load-bearing idea is the pipeline topology: Raw GNSS processing creates the representation needed by Phone position snapping, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Raw GNSS processing :red_circle:

This stage extract pseudo-ranges and carrier phases from raw Android log files (custom parsing).

### Outlier rejection :red_circle:

This stage filter satellite signals based on carrier-to-noise density (C/N0) and multipath indicators (pandas).

### State estimation :red_circle:

This stage apply an Extended Kalman Filter (EKF) to fuse the GNSS positioning with IMU data (accelerometer, gyro) (custom math).

### Smoothing :red_circle:

This stage apply a forward-backward Rauch-Tung-Striebel (RTS) smoother to the EKF trajectory (scipy).

### Phone position snapping :red_circle:

This stage snap the derived trajectory to a localized highway/street grid using map matching.
