## Pipeline Stages

### Problem reduction :red_circle:

This stage translate the string permutation problem structurally into a classic Asymmetric Traveling Salesman Problem (ATSP) (Python).

### Distance matrix generation :red_circle:

This stage compute the exact edge weights (overlap distance) between every possible mandatory substring (numpy).

### Greedy initialization :red_circle:

This stage run a fast greedy search to generate a mathematically valid, but non-optimal, baseline path through the graph (NetworkX).

### LKH Optimization :red_circle:

This stage pass the distance matrix and baseline path into LKH-3 (a state-of-the-art heuristic solver for ATSP) to iteratively optimize the route (LKH-3 C binary).

### Wildcard injection :red_circle:

This stage post-process the solver's output by algorithmically injecting the allowable "wildcard" characters to squeeze out the final few characters of string length (Python).
