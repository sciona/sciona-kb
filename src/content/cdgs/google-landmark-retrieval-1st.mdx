# Google Landmark Retrieval - 1st Place (2020) - Pipeline Template

> This template captures the ordered steps used by Google Landmark Retrieval - 1st Place (2020): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## When to Use This

- The new task has the same broad problem type: Image Retrieval / Metric Learning.
- The available data supports the same ordered transformations and intermediate artifacts as the documented winning pipeline.
- Offline batch processing is acceptable, including feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Image preprocessing creates the representation needed by Spatial verification, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Image preprocessing :red_circle:

This stage resize and centrally crop landmark images (OpenCV).

### Metric learning model :red_circle:

This stage train a ResNet-101 with a Generalized Mean (GeM) pooling layer and ArcFace loss to separate class embeddings radially (PyTorch).

### Embedding extraction :red_circle:

This stage extract 512-dimensional L2-normalized embeddings for all index and query images.

### Nearest neighbor search :white_check_mark:

This stage perform exact cosine similarity search to find the top 100 index images for every query image (FAISS).

**Bound atom:** `sciona.atoms.ml.sklearn.neighbors.nearest_neighbors_fit`

### Spatial verification :warning:

This stage run SuperPoint and RANSAC on the top retrieved pairs to rerank the candidates based on actual geometric feature matching (OpenCV/PyTorch).

**Bound atom:** `sciona.atoms.ml.sklearn.linear_model.ransac.ransac_consensus_is_better`
