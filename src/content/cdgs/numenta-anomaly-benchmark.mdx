## Key Insight

The load-bearing idea is the pipeline topology: Streaming ingestion creates the representation needed by Windowed Evaluation, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Streaming ingestion :red_circle:

This stage load single-variate time-series readings sequentially without future look-ahead (pandas).

### Representation generation :red_circle:

This stage use an isolation forest or continuous autoencoder to map the trailing window of points into a compressed feature space (sklearn/Keras).

### Predictive scoring :red_circle:

This stage calculate the deviation/reconstruction error between the incoming data point and the model's expectation (numpy).

### Dynamic thresholding :red_circle:

This stage maintain a rolling mean and variance of the anomaly scores, triggering an anomaly flag only when the current score exceeds $3\sigma$ of the local rolling window (numpy).

### Windowed Evaluation :red_circle:

This stage score the predictions using the NAB metric, which rewards early detection and heavily penalizes late detection within specific "anomaly windows".
