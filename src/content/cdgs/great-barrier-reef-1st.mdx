## Key Insight

The load-bearing idea is the pipeline topology: Video Frame Extraction creates the representation needed by Post-processing, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Video Frame Extraction :red_circle:

This stage load continuous MP4 videos, extract 1280x720 spatial frames, and split the dataset via a 3-fold cross-validation strategy specifically stratified by video_id to prevent temporal leakage (cv2, pandas).

### Patch Generation :red_circle:

This stage slice the original high-resolution frames into smaller 512x320 pixel patches to artificially enlarge small starfish targets for the detector (numpy).

### Boundary Filtering :red_circle:

This stage remove any bounding boxes that intersect the patch boundaries to prevent the network from learning truncated artifacts.

### Data Augmentation :warning:

This stage apply geometric rotation, mixup blending, and bounding-box-safe augmentations (albumentations).

**Bound atom:** `sciona.atoms.dl.image_augmentation.mixup_apply`

### Detector Training :red_circle:

This stage train an array of YOLOv5 models on the isolated patches utilizing modified, highly permissive confidence thresholds (box=0.2, iou_t=0.3) to maximize recall (torch, yolov5).

### Full-Frame Inference :red_circle:

This stage infer bounding box coordinates on the original unpatched 1280x720 test images.

### Classification Rescoring :red_circle:

This stage extract the predicted bounding box crops and pass them through a completely independent secondary CNN classifier to penalize and discard false positive predictions (torch.nn, timm).

### Post-processing :red_circle:

This stage apply Non-Maximum Suppression (NMS) to collapse overlapping boxes and format final predictions for submission (torchvision.ops).
