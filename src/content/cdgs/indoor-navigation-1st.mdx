# Indoor Location & Navigation 1st Place — Wi-Fi + IMU Sensor Fusion (2021)

> Predict indoor location by combining Wi-Fi signal strength measurements with motion sensor data (accelerometer and gyroscope), blending the two estimates with a Kalman filter, and snapping the result onto known hallway paths.

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.indoor_navigation_1st" readOnly={true} /> */}

## When to Use This

- Two or more sensor modalities provide complementary positioning signals: one absolute but noisy (e.g., Wi-Fi RSSI, BLE beacons, GPS) and one relative but drifting (e.g., IMU/PDR, odometry, visual odometry)
- Known environment geometry (floor plans, road networks, building layouts) is available for post-hoc trajectory correction
- Positioning must be continuous over a trajectory, not just point estimates — the temporal structure of movement constrains predictions between observation points
- Wi-Fi infrastructure is dense enough to provide meaningful fingerprinting signal (multiple visible APs per scan)

## Key Insight

The Kalman filter fuses absolute (Wi-Fi, noisy but unbiased) with relative (IMU/PDR, precise but drifting) positioning — this is the classic complementary sensor fusion pattern. Wi-Fi provides coarse global corrections that prevent PDR drift from accumulating, while PDR provides smooth local trajectory structure between sparse Wi-Fi observations. Neither source alone achieves the accuracy of the fusion.

## Pipeline Stages

### Wi-Fi Feature Extraction :red_circle:

Organize the Wi-Fi signal strength data into a structured table where each row is a moment in time and each column is a specific Wi-Fi access point, with the signal strength as the value.

### Sensor Alignment :white_check_mark:

Synchronize the fast motion sensor readings with the slower Wi-Fi measurements by interpolating the motion data to match Wi-Fi measurement times.

**Bound atom:** `sciona.atoms.time_series_features.interpolate_to_timestamps`

### Wi-Fi Position Model :red_circle:

Train machine learning models to estimate location from the pattern of Wi-Fi signal strengths, using both a tree-based model and a neural network for complementary predictions.

### Pedestrian Dead Reckoning :warning:

Figure out how far and in which direction the person walked between each Wi-Fi measurement by counting steps from the accelerometer and tracking turning from the gyroscope.

**Bound atom:** `sciona.atoms.geo.geospatial_sensors.detect_steps`

### Kalman Sensor Fusion :warning:

Blend the Wi-Fi-based location estimates (accurate but noisy) with the step-based movement estimates (smooth but drifting) using a statistical filter that optimally weights each source based on its reliability.

**Bound atom:** `sciona.atoms.robotics.pronto.dynamic_stance_estimator.update_step`

### Snap to Grid :warning:

**Bound atom:** `sciona.atoms.geo.geospatial_sensors.snap_to_nearest`
