## Key Insight

The load-bearing idea is the pipeline topology: Voxel Preprocessing creates the representation needed by Multi-Objective Blending, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Voxel Preprocessing :red_circle:

This stage load massive 3D medical scans and resize them via interpolation to a uniform, memory-efficient 128x128x128 volume (nibabel, scipy.ndimage).

### 3D Semantic Segmentation :white_check_mark:

This stage train a 3D U-Net (utilizing a ResNet18d or EfficientNetV2s backbone) to segment the C1-C7 vertebrae into 7 distinct spatial channels (monai, torch).

**Bound atom:** `sciona.atoms.dl.detection.coordinate_aware_3d_unet`

### Mask Generation :red_circle:

This stage predict and cache the 3D masks for all 2,000 samples in the training set using the trained segmentation network.

### 2.5D Cropping :red_circle:

This stage extract individual 2D slice sequences for each localized vertebra based precisely on the generated 3D masks, reducing the computational burden for the next phase (numpy).

### Sequential Classification :red_circle:

This stage pass the sequential 2.5D slices through a CNN+LSTM classifier to detect micro-fractures across the spatial slices (torch.nn.LSTM, timm).

### Multi-Objective Blending :red_circle:

This stage linearly combine the final probabilistic predictions from two different types of Stage-2 classification architectures.
