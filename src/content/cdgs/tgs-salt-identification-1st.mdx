# TGS Salt Identification Challenge - 1st Place (2018) - Pipeline Template

> This template captures the ordered steps used by TGS Salt Identification Challenge - 1st Place (2018): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

- The new task has the same broad problem type: Image Segmentation (Seismic Data).

- Offline processing is acceptable, including search, feature extraction, model ensembling, or post-processing stages.

## Key Insight

The load-bearing idea is the pipeline topology: Depth feature fusion creates the representation needed by Ensembling with TTA, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Depth feature fusion :red_circle:

This stage explicitly broadcast the 1D depth coordinate of the seismic slice across the entire X-axis and concatenate it as an additional channel to the 2D seismic image (numpy).

### U-Net Backbone :red_circle:

This stage train a U-Net with a ResNet34 backbone on the augmented 2D slices (PyTorch).

### Spatial SE Block :red_circle:

This stage implement custom Spatial Squeeze-and-Excitation blocks to allow the network to weigh specific spatial regions over others (PyTorch).

### Lovász-Softmax Loss :red_circle:

This stage initialize training with BCE loss, then switch entirely to Lovász-Softmax loss directly optimizing the Intersection-over-Union (IoU) metric without mathematically smooth approximations.

### Ensembling with TTA :red_circle:

This stage sum the mask probabilities from 5-fold cross-validation utilizing Test-Time Augmentation (horizontal flips) and threshold at 0.5.
