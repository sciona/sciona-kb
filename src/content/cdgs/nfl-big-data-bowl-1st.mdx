## Key Insight

The load-bearing idea is the pipeline topology: Spatial normalization creates the representation needed by Continuous Ranked Probability Score (CRPS), so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Spatial normalization :red_circle:

This stage strictly flip all player X/Y coordinates and velocities so the offensive team is always moving left-to-right (numpy).

### Relative coordinate mapping :white_check_mark:

This stage recalculate all 21 players' X/Y positions relative to the exact X/Y position of the ball carrier (pandas).

**Bound atom:** `sciona.atoms.cs.game_ai.relative_polar_coordinate_transform`

### Spatial tensor formatting :red_circle:

This stage flatten the 22 players' features into a 1D sequence and sort them deterministically (e.g., offense first, defense second, sorted by distance to ball) (numpy).

### 1D CNN Architecture :red_circle:

This stage pass the 1D sequence through a 1D Convolutional Neural Network (instead of an MLP) to naturally capture localized spatial relationships between adjacent players (Keras).

### Continuous Ranked Probability Score (CRPS) :red_circle:

This stage output a cumulative distribution function (CDF) of expected yards gained, strictly optimizing for CRPS loss (Keras).
