## Key Insight

The load-bearing idea is the pipeline topology: Prompt injection creates the representation needed by Majority Voting, so later stages optimize the metric without solving raw-data routing from scratch.

## Pipeline Stages

### Prompt injection :red_circle:

This stage wrap the math problem in a system prompt instructing the LLM to write a Python script to solve it algebraically (HuggingFace).

### Code Generation :red_circle:

This stage run a fine-tuned Code Llama or DeepSeek-Math LLM to generate the raw Python code (vLLM).

### Sandbox Execution :red_circle:

This stage execute the generated Python code securely in an isolated subprocess (Python subprocess).

### Reflection Loop :red_circle:

This stage if the code errors, feed the traceback directly back into the LLM prompt and ask it to fix the bug (up to 3 retries).

### Majority Voting :red_circle:

This stage run the generation/execution loop 50 times at high temperature, collect all successful integer outputs, and submit the mode (most common answer) (numpy).
