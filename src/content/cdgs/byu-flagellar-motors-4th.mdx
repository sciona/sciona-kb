## Key Insight

The load-bearing idea is the pipeline topology: Data Ingestion creates the representation needed by Ensembling, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Data Ingestion :red_circle:

This stage load cryo-ET biological imagery and systematically merge it with supplementary external datasets to increase diversity (pandas).

### Medical Augmentation :red_circle:

This stage apply domain-specific tensor augmentations (rotation, volumetric scaling, Gaussian noise injection) tailored specifically for biological and medical imaging (MONAI).

### Classification Feature Extraction :red_circle:

This stage train a straightforward 3D-ResNet18 classification backbone to simply identify whether regions contain flagellar motors, acting as a rapid filtration mechanism (torch.nn).

### Object Detection :red_circle:

This stage train specialized medical object detection networks strictly on the high-probability regions identified by the ResNet (MONAI).

### Model Export & Acceleration :red_circle:

This stage compile the trained models to TensorRT via TorchScript (JIT). This architectural choice enables ultra-fast inference, deliberately sacrificing a fraction of precision to allow for a vastly larger ensemble within strict competition time limits (tensorrt, torch.jit).

### Ensembling :red_circle:

This stage average the resulting bounding box coordinates and confidences across the parallel ResNet and MONAI detection streams (numpy).
