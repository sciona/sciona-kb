## Key Insight

The load-bearing idea is the pipeline topology: Sequence Tokenization creates the representation needed by Decoding, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

## Pipeline Stages

### Sequence Tokenization :white_check_mark:

This stage tokenize raw text data and generate corresponding index offsets (transformers.AutoTokenizer).

**Bound atom:** `sciona.atoms.ml.tokenizer.tokenize`

### QA Reformulation :red_circle:

This stage reformat the standard NER data arrays entirely into a Question-Answering structure, guaranteeing one absolute "onset" tag and one "wakeup" tag per continuous train sequence.

### QA Softmax Head :red_circle:

This stage instead of utilizing a standard BertForTokenClassification module (which acts as an NLP analog to image segmentation), structurally attach a BertForQuestionAnswering head (the NLP analog to bounding-box object detection) to the core transformer backbone (transformers, torch.nn).

### Span Prediction :red_circle:

This stage train the reformulated model to predict explicit start and end logit spans across the sequence rather than classifying every individual token sequentially.

### Decoding :warning:

This stage convert the predicted start/end probabilistic indices back to highly accurate text spans utilizing the saved offsets (numpy).

**Bound atom:** `sciona.atoms.ml.text_nlp.bio_decode`
