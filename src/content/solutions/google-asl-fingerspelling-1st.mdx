# Google American Sign Language Fingerspelling - 1st Place (2023) - Pipeline Template

## Problem

Sequence-to-Sequence (Spatial Landmarks to Text) solution template for Google American Sign Language Fingerspelling - 1st Place (2023), modeled as a 5-stage deterministic pipeline: Landmark interpolation, Affine Augmentation, Feature engineering, CNN + Transformer Architecture, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by Google American Sign Language Fingerspelling - 1st Place (2023): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

1. **Landmark interpolation** -- This stage fill dropped MediaPipe 3D coordinate frames using linear interpolation to ensure a continuous time series (pandas).
1. **Affine Augmentation** -- This stage apply random 3D rotations, scaling, and sheer to the skeletal coordinates to simulate different camera angles during training (numpy).
1. **Feature engineering** -- This stage calculate Euclidean distance, velocity, and angles between specific finger joints (thumb to index, wrist to pinky) explicitly (numpy).
1. **CNN + Transformer Architecture** -- This stage pass the coordinates through a 1D CNN to extract local temporal features, followed by a Transformer Encoder for global sequence context (PyTorch).
1. **CTC Loss Translation** -- This stage decode the sequence into raw text tokens using Connectionist Temporal Classification (CTC) loss to handle the misalignment between sequence length and text length (PyTorch).

## Key Insight

The load-bearing idea is the pipeline topology: Landmark interpolation creates the representation needed by CTC Loss Translation, so later stages optimize the metric without solving raw-data routing from scratch.
