# Cdiscount Image Classification - 1st Place (2017) - Pipeline Template

## Problem

Extreme Classification (10M+ Images, 5000+ Classes) solution template for Cdiscount Image Classification - 1st Place (2017), modeled as a 5-stage deterministic pipeline: BSON Chunking, Multi-GPU Data Loader, CNN Backbone, Hierarchical Classification Head, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by Cdiscount Image Classification - 1st Place (2017): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.cdiscount_image_classification_1st" readOnly={true} /> */}

1. **BSON Chunking** -- This stage parse the massive 50GB BSON file line-by-line without loading into RAM, decoding images to JPEG and writing to a lightning-fast LMDB database (PyMongo/LMDB).
1. **Multi-GPU Data Loader** -- This stage write a custom multi-threaded data loader utilizing PyTorch's DistributedDataParallel to constantly feed 8 GPUs without I/O bottlenecks (PyTorch).
1. **CNN Backbone** -- This stage train ResNet50 and InceptionV3 backbones utilizing a heavily tuned learning rate scheduler (PyTorch).
1. **Hierarchical Classification Head** -- This stage replace the standard flat softmax with a hierarchical softmax predicting Category -> SubCategory -> Item, reducing the vast 5000-class search space (custom PyTorch logic).
1. **Test-Time Augmentation (TTA)** -- This stage run 4x spatial crops on the test images, sum the log-probabilities, and select the argmax class.

## Key Insight

The load-bearing idea is the pipeline topology: BSON Chunking creates the representation needed by Test-Time Augmentation (TTA), so later stages optimize the metric without solving raw-data routing from scratch.

## Failure Modes

- Distribution shift breaks handcrafted thresholds, validation-derived weights, or post-processing rules.
- Missing metadata prevents faithful construction of candidates, groups, windows, graphs, or cross-validation folds.
- A weak first-stage recall, search, or representation step cannot be fully recovered by downstream rescoring or ensembling.
