# OGB Large-Scale Challenge - MAG240M - 1st Place (2021) - Pipeline Template

## Problem

Graph ML / Node Classification solution template for OGB Large-Scale Challenge - MAG240M - 1st Place (2021), modeled as a 5-stage deterministic pipeline: Node text extraction, Graph sampling, Message passing, Label Propagation, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by OGB Large-Scale Challenge - MAG240M - 1st Place (2021): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

1. **Node text extraction** -- This stage run RoBERTa across all 240 million paper nodes to generate dense 768-dimensional text embeddings (HuggingFace).
1. **Graph sampling** -- This stage construct a heterogeneous neighborhood sampling pipeline to handle the massive graph size without encountering CPU out-of-memory errors (DGL).
1. **Message passing** -- This stage train a multi-layer GraphSAGE architecture, passing the RoBERTa embeddings iteratively through the citation edges (PyTorch / DGL).
1. **Label Propagation** -- This stage execute a classic Label Propagation Algorithm (LPA) over the graph's known labels to generate a structural baseline score (NetworkX).
1. **Blending** -- This stage dynamically combine the deep GraphSAGE predictions with the structural LPA predictions using a learned weight array (numpy).

## Key Insight

The load-bearing idea is the pipeline topology: Node text extraction creates the representation needed by Blending, so later stages optimize the metric without solving raw-data routing from scratch.
