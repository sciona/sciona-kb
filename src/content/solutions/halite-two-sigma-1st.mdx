# Halite by Two Sigma - 1st Place (2020) - Pipeline Template

## Problem

Reinforcement Learning / Board Games solution template for Halite by Two Sigma - 1st Place (2020), modeled as a 5-stage deterministic pipeline: Top-bot scraping, Value Network training, Policy Network training, Monte Carlo Tree Search (MCTS), and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by Halite by Two Sigma - 1st Place (2020): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.halite_two_sigma_1st" readOnly={true} /> */}

1. **Top-bot scraping** -- This stage constantly scrape replays of the top 50 bots off the competition leaderboard to serve as dynamic training data (requests).
1. **Value Network training** -- This stage train a ResNet architecture to input the Halite board state and output the final expected win probability of the game (PyTorch).
1. **Policy Network training** -- This stage train a separate architecture to predict the exact next move of the expert bots (Behavioral Cloning).
1. **Monte Carlo Tree Search (MCTS)** -- This stage implement a highly optimized MCTS in a low-level language. Use the Policy network to guide the search tree and the Value network to evaluate terminal states (custom C++).
1. **Inference compiling** -- This stage compile the C++ MCTS algorithm into a .so file executable by the Python competition environment.

## Key Insight

The load-bearing idea is the pipeline topology: Top-bot scraping creates the representation needed by Inference compiling, so later stages optimize the metric without solving raw-data routing from scratch.

## Failure Modes

- Distribution shift breaks handcrafted thresholds, validation-derived weights, or post-processing rules.
- Missing metadata prevents faithful construction of candidates, groups, windows, graphs, or cross-validation folds.
- A weak first-stage recall, search, or representation step cannot be fully recovered by downstream rescoring or ensembling.
