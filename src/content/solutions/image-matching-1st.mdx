# Image Matching Challenge 1st Place — Fundamental Matrix from Local Feature Matching (2022)

## Problem

Multi-method local feature extraction and matching pipeline combining SuperPoint+SuperGlue sparse matching with LoFTR dense matching, fused through MAGSAC++ robust estimation to recover the fundamental matrix.

## Approach

Find matching points between two photos using multiple neural-network detectors, combine all the matches, then use a robust algorithm to throw out wrong matches and compute the geometric relationship between the two camera views.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.image_matching_1st" readOnly={true} /> */}

1. **Dynamic Resizing** -- Resize unusually shaped images (very tall or very wide) so that feature detection networks can process them reliably.
1. **SuperPoint Extraction** -- Use a neural network to find distinctive points in each image and describe each point with a numerical fingerprint for later matching.
1. **LoFTR Extraction** -- Use a different neural network that directly finds matching points between two images at once, producing many more matches than the sparse detector.
1. **SuperGlue Matching** -- Use a neural network to figure out which distinctive points in image 1 correspond to which distinctive points in image 2, using the numerical fingerprints from SuperPoint.
1. **Match Concatenation** -- Merge all the matched point pairs from both methods into one big list so the geometry estimator can work with all of them together.
1. **Fundamental Matrix Estimation** -- Use a robust algorithm to figure out which matched points are correct and which are wrong, then compute the geometric relationship (fundamental matrix) between the two camera views from only the correct matches.

## Key Insight

Using BOTH sparse (SuperPoint/SuperGlue) and dense (LoFTR) matchers in parallel, then concatenating their correspondences before MAGSAC++, covers both textured and textureless regions. Neither matcher alone achieves full coverage: sparse methods miss low-texture areas, dense methods produce noisier matches in high-texture areas. The combined pool gives MAGSAC++ a richer inlier set to work with.

## Failure Modes

- Scenes with no visual overlap between images: both matchers produce only outliers, and MAGSAC++ returns a degenerate fundamental matrix
- Purely planar scenes: fundamental matrix is undefined (infinite epipoles), and the pipeline should use homography estimation instead
- Very small images (&lt;256px): both SuperPoint and LoFTR produce too few features for reliable geometry estimation
