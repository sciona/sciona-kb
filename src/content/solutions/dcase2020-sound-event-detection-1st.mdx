# DCASE 2020 Task 4 (Sound Event Detection) - 1st Place (2020) - Pipeline Template

## Problem

Audio SED / Semi-supervised solution template for DCASE 2020 Task 4 (Sound Event Detection) - 1st Place (2020), modeled as a 5-stage deterministic pipeline: Audio conversion, Mean-Teacher framework initialization, Supervised phase, Unsupervised consistency phase, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by DCASE 2020 Task 4 (Sound Event Detection) - 1st Place (2020): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.dcase2020_sound_event_detection_1st" readOnly={true} /> */}

1. **Audio conversion** -- This stage generate standard Log-Mel Spectrograms from both labeled and weakly-labeled datasets (librosa).
1. **Mean-Teacher framework initialization** -- This stage instantiate two identical CNN-RNN architectures (CRNNs): one designated "Student", one designated "Teacher" (PyTorch).
1. **Supervised phase** -- This stage train the Student model normally on the small set of ground-truth labeled audio samples.
1. **Unsupervised consistency phase** -- This stage pass unlabeled audio through both models. Use MSE loss to force the Student model's predictions to match the Teacher model's predictions, constantly updating the Teacher via Exponential Moving Average (EMA) of the Student's weights (PyTorch).
1. **Post-processing** -- This stage apply a mathematical Median Filter to the frame-level predictions to eliminate impossible rapid clicking/switching between active sound classes (scipy.signal).

## Key Insight

The load-bearing idea is the pipeline topology: Audio conversion creates the representation needed by Post-processing, so later stages optimize the metric without solving raw-data routing from scratch.

## Failure Modes

- Distribution shift breaks handcrafted thresholds, validation-derived weights, or post-processing rules.
- Missing metadata prevents faithful construction of candidates, groups, windows, graphs, or cross-validation folds.
- A weak first-stage recall, search, or representation step cannot be fully recovered by downstream rescoring or ensembling.
