# NFL Big Data Bowl - 1st Place (2020) - Pipeline Template

## Problem

Spatial Trajectory / 1D CNN solution template for NFL Big Data Bowl - 1st Place (2020), modeled as a 5-stage deterministic pipeline: Spatial normalization, Relative coordinate mapping, Spatial tensor formatting, 1D CNN Architecture, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by NFL Big Data Bowl - 1st Place (2020): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.nfl_big_data_bowl_1st" readOnly={true} /> */}

1. **Spatial normalization** -- This stage strictly flip all player X/Y coordinates and velocities so the offensive team is always moving left-to-right (numpy).
1. **Relative coordinate mapping** -- This stage recalculate all 21 players' X/Y positions relative to the exact X/Y position of the ball carrier (pandas).
1. **Spatial tensor formatting** -- This stage flatten the 22 players' features into a 1D sequence and sort them deterministically (e.g., offense first, defense second, sorted by distance to ball) (numpy).
1. **1D CNN Architecture** -- This stage pass the 1D sequence through a 1D Convolutional Neural Network (instead of an MLP) to naturally capture localized spatial relationships between adjacent players (Keras).
1. **Continuous Ranked Probability Score (CRPS)** -- This stage output a cumulative distribution function (CDF) of expected yards gained, strictly optimizing for CRPS loss (Keras).

## Key Insight

The load-bearing idea is the pipeline topology: Spatial normalization creates the representation needed by Continuous Ranked Probability Score (CRPS), so later stages optimize the metric without solving raw-data routing from scratch.
