# Optiver Realized Volatility Prediction - 1st Place (2021) - Pipeline Template

## Problem

Financial Time Series (High-Frequency Limit Order Book) solution template for Optiver Realized Volatility Prediction - 1st Place (2021), modeled as a 5-stage deterministic pipeline: LOB parsing, Time-bucketing, Feature clustering, Cross-sectional aggregation, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by Optiver Realized Volatility Prediction - 1st Place (2021): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

1. **LOB parsing** -- This stage compute the Weighted Average Price (WAP) and log returns continuously across the raw bid/ask tick data (pandas).
1. **Time-bucketing** -- This stage aggregate the tick-level data into fixed historical time windows (e.g., last 600 seconds, 300 seconds, 100 seconds) computing realized volatility per bucket (numpy).
1. **Feature clustering** -- This stage cluster different financial stocks based on their historical volatility correlations to create "market sector" proxy features (scikit-learn KMeans).
1. **Cross-sectional aggregation** -- This stage compute the mean volatility of the entire market at each specific time bucket to contextualize individual stock movements (pandas).
1. **Custom Objective GBDT** -- This stage train LightGBM models utilizing a custom Root Mean Square Percentage Error (RMSPE) loss function directly aligning with the competition metric (LightGBM).

## Key Insight

The load-bearing idea is the pipeline topology: LOB parsing creates the representation needed by Custom Objective GBDT, so later stages optimize the metric without solving raw-data routing from scratch.
