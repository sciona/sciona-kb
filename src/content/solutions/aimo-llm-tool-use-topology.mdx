# AI Mathematical Olympiad (AIMO) Prize - Top Solution Topology (2024) - Pipeline Template

## Problem

Automated Reasoning / LLM Tool Use solution template for AI Mathematical Olympiad (AIMO) Prize - Top Solution Topology (2024), modeled as a 5-stage deterministic pipeline: Prompt injection, Code Generation, Sandbox Execution, Reflection Loop, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by AI Mathematical Olympiad (AIMO) Prize - Top Solution Topology (2024): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.aimo_llm_tool_use_topology" readOnly={true} /> */}

1. **Prompt injection** -- This stage wrap the math problem in a system prompt instructing the LLM to write a Python script to solve it algebraically (HuggingFace).
1. **Code Generation** -- This stage run a fine-tuned Code Llama or DeepSeek-Math LLM to generate the raw Python code (vLLM).
1. **Sandbox Execution** -- This stage execute the generated Python code securely in an isolated subprocess (Python subprocess).
1. **Reflection Loop** -- This stage if the code errors, feed the traceback directly back into the LLM prompt and ask it to fix the bug (up to 3 retries).
1. **Majority Voting** -- This stage run the generation/execution loop 50 times at high temperature, collect all successful integer outputs, and submit the mode (most common answer) (numpy).

## Key Insight

The load-bearing idea is the pipeline topology: Prompt injection creates the representation needed by Majority Voting, so later stages optimize the metric without solving raw-data routing from scratch.
