# APTOS 2019 Blindness Detection - 1st Place (2019) - Pipeline Template

## Problem

Ordinal Classification (Medical Imaging) solution template for APTOS 2019 Blindness Detection - 1st Place (2019), modeled as a 5-stage deterministic pipeline: Image standardization, CNN Backbone, Quadratic Weighted Kappa (QWK) Loss, Pre-training & Fine-tuning, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by APTOS 2019 Blindness Detection - 1st Place (2019): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.aptos_blindness_1st" readOnly={true} /> */}

1. **Image standardization** -- This stage apply Ben Graham's preprocessing method: crop to the circular retina, subtract local median color to highlight blood vessels, and resize (OpenCV).
1. **CNN Backbone** -- This stage train EfficientNet and SE-ResNeXt architectures initialized with ImageNet weights (PyTorch).
1. **Quadratic Weighted Kappa (QWK) Loss** -- This stage optimize the network using a smooth QWK loss function, heavily penalizing off-by-2 errors (e.g., predicting 0 when truth is 4) more than off-by-1 errors (custom PyTorch loss).
1. **Pre-training & Fine-tuning** -- This stage pre-train the model on massive public datasets from past 2015 competitions, then strictly fine-tune only the last 2 layers on the smaller 2019 competition dataset.
1. **Threshold Calibration** -- This stage use Nelder-Mead optimization to perfectly tune the decision boundaries between classes [0, 1, 2, 3, 4] to maximize the QWK metric.

## Key Insight

The load-bearing idea is the pipeline topology: Image standardization creates the representation needed by Threshold Calibration, so later stages optimize the metric without solving raw-data routing from scratch.
