# IceCube - Neutrinos in Deep Ice - 1st Place (2023) - Pipeline Template

## Problem

3D Graph / Sequence Regression (Directional) solution template for IceCube - Neutrinos in Deep Ice - 1st Place (2023), modeled as a 5-stage deterministic pipeline: Point cloud processing, Graph definition, Graph Neural Network, Transformer block, and downstream competition-specific post-processing.

## Approach

This template captures the ordered steps used by IceCube - Neutrinos in Deep Ice - 1st Place (2023): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## CDG Walkthrough

1. **Point cloud processing** -- This stage format detector hits into node features (x, y, z, time, charge) (numpy).
1. **Graph definition** -- This stage build a KNN graph connecting hits based strictly on spatial and temporal proximity (PyTorch Geometric).
1. **Graph Neural Network** -- This stage pass the graph through a DynEdge architecture (GraphNet) to allow message passing between detector nodes (PyTorch).
1. **Transformer block** -- This stage feed the pooled graph node embeddings into a transformer encoder for global context mapping.
1. **Directional loss** -- This stage predict 3D spherical coordinates using a von Mises-Fisher (vMF) loss function to handle angular periodicity constraints.

## Key Insight

The load-bearing idea is the pipeline topology: Point cloud processing creates the representation needed by Directional loss, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.
