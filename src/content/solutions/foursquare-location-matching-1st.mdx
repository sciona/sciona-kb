# Foursquare - Location Matching - 1st Place (2022) - Pipeline Template

## Problem

Tabular+Text Matching / Graph formulation solution template for Foursquare - Location Matching - 1st Place (2022), modeled as a 5-stage deterministic pipeline: Candidate generation, Feature extraction, Binary classification, Graph creation, and downstream competition-specific post-processing.

## Approach

This template captures the ordered steps used by Foursquare - Location Matching - 1st Place (2022): prepare the data, build task-specific representations, train or apply the main model, and convert outputs into the final competition submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.foursquare_location_matching_1st" readOnly={true} /> */}

1. **Candidate generation** -- This stage find the K-nearest spatial neighbors for each location using Haversine distance, and K-nearest text neighbors based on name (FAISS/sklearn).
1. **Feature extraction** -- This stage compute string similarity scores (Levenshtein, Jaro-Winkler) and distance metrics for all generated candidate pairs (rapids/cuml).
1. **Binary classification** -- This stage train an XGBoost model to predict "match" or "no match" for every pair.
1. **Graph creation** -- This stage treat predictions as edges and build a graph of matched locations (NetworkX).
1. **Connected components** -- This stage extract discrete clusters from the graph and assign identical IDs to all nodes within a cluster (NetworkX).

## Key Insight

The load-bearing idea is the pipeline topology: Candidate generation creates the representation needed by Connected components, so later stages can optimize the competition metric without solving raw-data ingestion from scratch.

- A weak first-stage recall step cannot be recovered by downstream rescoring or ensembling.
