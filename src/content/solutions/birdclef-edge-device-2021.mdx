# BirdCLEF (Edge Device Track) - High-Tier Solution (2021) - Pipeline Template

## Problem

Edge ML / Audio Classification (Constrained) solution template for BirdCLEF (Edge Device Track) - High-Tier Solution (2021), modeled as a 5-stage deterministic pipeline: Audio feature extraction, Teacher Model Training, Student Model Training (Distillation), TFLite Conversion, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by BirdCLEF (Edge Device Track) - High-Tier Solution (2021): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.birdclef_edge_device_2021" readOnly={true} /> */}

1. **Audio feature extraction** -- This stage generate standard Mel Spectrograms from continuous audio files (librosa).
1. **Teacher Model Training** -- This stage train a massive, high-accuracy ResNeSt-50 model on the spectrograms without any hardware constraints (PyTorch).
1. **Student Model Training (Distillation)** -- This stage train a micro MobileNetV3 architecture using the soft probability labels output by the Teacher model rather than the hard ground-truth labels (PyTorch).
1. **TFLite Conversion** -- This stage convert the trained PyTorch MobileNetV3 into an ONNX format, then strictly quantize it to INT8 precision using TensorFlow Lite to ensure sub-50ms inference times (ONNX/TFLite).
1. **On-device Thresholding** -- This stage implement fixed threshold bounds inside the TFLite wrapper to filter predictions before final return.

## Key Insight

The load-bearing idea is the pipeline topology: Audio feature extraction creates the representation needed by On-device Thresholding, so later stages optimize the metric without solving raw-data routing from scratch.
