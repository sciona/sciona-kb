# Indoor Location & Navigation 1st Place — Wi-Fi + IMU Sensor Fusion (2021)

## Problem

Indoor positioning pipeline fusing Wi-Fi fingerprint-based position estimates (LightGBM + 1D CNN) with IMU-based pedestrian dead reckoning via Kalman filtering and cost minimization, with final trajectory snapping to known walkway geometry.

## Approach

Predict indoor location by combining Wi-Fi signal strength measurements with motion sensor data (accelerometer and gyroscope), blending the two estimates with a Kalman filter, and snapping the result onto known hallway paths.

## CDG Walkthrough

1. **Wi-Fi Feature Extraction** -- Organize the Wi-Fi signal strength data into a structured table where each row is a moment in time and each column is a specific Wi-Fi access point, with the signal strength as the value.
1. **Sensor Alignment** -- Synchronize the fast motion sensor readings with the slower Wi-Fi measurements by interpolating the motion data to match Wi-Fi measurement times.
1. **Wi-Fi Position Model** -- Train machine learning models to estimate location from the pattern of Wi-Fi signal strengths, using both a tree-based model and a neural network for complementary predictions.
1. **Pedestrian Dead Reckoning** -- Figure out how far and in which direction the person walked between each Wi-Fi measurement by counting steps from the accelerometer and tracking turning from the gyroscope.
1. **Kalman Sensor Fusion** -- Blend the Wi-Fi-based location estimates (accurate but noisy) with the step-based movement estimates (smooth but drifting) using a statistical filter that optimally weights each source based on its reliability.
1. **Snap to Grid** -- Adjust the predicted path so that each point lies on a known hallway or walkway, since people can only walk where there are actual corridors.

## Key Insight

The Kalman filter fuses absolute (Wi-Fi, noisy but unbiased) with relative (IMU/PDR, precise but drifting) positioning — this is the classic complementary sensor fusion pattern. Wi-Fi provides coarse global corrections that prevent PDR drift from accumulating, while PDR provides smooth local trajectory structure between sparse Wi-Fi observations. Neither source alone achieves the accuracy of the fusion.

## Failure Modes

- Sparse Wi-Fi coverage: if only 1-2 APs are visible per scan, Wi-Fi fingerprinting degenerates to coarse room-level localization, and the Kalman filter trusts these noisy estimates too much
- Stationary periods: PDR produces zero displacement when the user is not walking, but Wi-Fi noise continues to perturb the fused estimate. The Kalman filter needs explicit stationary detection to avoid position jitter
- Multi-floor transitions: elevator/staircase movement violates the 2D PDR model and can cause the Kalman filter to diverge until the Wi-Fi signal re-anchors on the correct floor
