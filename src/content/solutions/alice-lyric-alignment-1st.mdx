# ALiCE - Automatic Lyric Alignment - 1st Place (2024) - Pipeline Template

## Problem

Audio-Text Alignment (Dynamic Time Warping) solution template for ALiCE - Automatic Lyric Alignment - 1st Place (2024), modeled as a 5-stage deterministic pipeline: Vocal isolation, Phoneme extraction, Text processing, Forced Alignment, and downstream competition-specific finalization.

## Approach

This template captures the ordered steps used by ALiCE - Automatic Lyric Alignment - 1st Place (2024): prepare the data, build task-specific representations, run the main modeling or search loop, and convert outputs into the final benchmark submission.

## CDG Walkthrough

{/* <ScionaViewerPlaceholder cdgId="solution.kaggle.alice_lyric_alignment_1st" readOnly={true} /> */}

1. **Vocal isolation** -- This stage pass raw audio through HTDemucs to strictly isolate the vocal track from the instrumental background (Demucs/torchaudio).
1. **Phoneme extraction** -- This stage pass the isolated vocals through an ASR Acoustic Model (Wav2Vec2) to extract frame-level phoneme probability matrices (HuggingFace).
1. **Text processing** -- This stage translate the provided raw lyric text into phonetic strings using an external grapheme-to-phoneme dictionary (g2p_en).
1. **Forced Alignment** -- This stage apply forced Dynamic Time Warping (DTW) to strictly align the predicted frame-level phoneme probabilities with the true textual phoneme sequence (torchaudio / custom DTW).
1. **Post-processing** -- This stage smooth the alignment boundaries using a median filter and assign exact start/end timestamps to every word.

## Key Insight

The load-bearing idea is the pipeline topology: Vocal isolation creates the representation needed by Post-processing, so later stages optimize the metric without solving raw-data routing from scratch.
